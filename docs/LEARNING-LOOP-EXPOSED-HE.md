# חשיפת לולאת הלמידה ומבחן עומס — סיכום

## 1. חשיפת "המוח" בממשק (Learning Matrix)

### דף התובנות (Insights)
- **עבור כל תחזית שהוערכה** (`status === 'evaluated'`) מוצג כעת בלוק בולט בעברית:
  - **כותרת:** "לקחים שנלמדו / מסקנות AI"
  - **תוכן:** `error_report` (מסקנת הלמידה שנוצרה על ידי ה-AI לאחר כישלון) ו־`learning_context` (הקשר הלמידה).
- הבלוק מעוצב עם רקע ענבר עדין ומסגרת, כך שניתן לקרוא במפורש מה ה-AI למד מכל כישלון.

### איפה לראות
- **תובנות AI** (`/insights`): גלול לרשומות תחזיות; בתחזיות שהוערכו (לאחר הרצת "הערך תחזיות ממתינות") יופיע הסעיף "לקחים שנלמדו / מסקנות AI" עם הטקסט המלא.

---

## 2. כפתור הערכה ידנית (Stress Test)

### לוח הבקרה (Ops)
- נוסף כפתור **"הערך תחזיות ממתינות"** בעמוד האופס (`/ops`), מעל בלוק מדדי המערכת.
- **פעולה:** לחיצה על הכפתור מפעילה את `evaluatePendingPredictions`: משווה תחזיות ממתינות למחירי שוק נוכחיים (Binance), מעדכן תוצאה (הצלחה/כישלון), מייצר הערות למידה (error_report) דרך Gemini, שומר ל־`historical_predictions` ומרענן את הדף.
- **משוב:** לאחר ההערכה מוצגת הודעה (למשל "הוערכו X תחזיות. רענן את הדף כדי לראות את הלקחים") והדף מתעדכן אוטומטית.

### איך לבדוק באתר החי
1. היכנס ל־**לוח הבקרה** (`/ops`).
2. לחץ על **"הערך תחזיות ממתינות"**.
3. חכה לסיום (כפתור "מעריך…" ואז הודעת הצלחה).
4. עבור ל־**תובנות AI** (`/insights`) או רענן את האופס — תראה בתחזיות שהוערכו את הסעיף "לקחים שנלמדו / מסקנות AI" עם המסקנות שנוצרו.
5. הרץ ניתוח חדש — ההקשר הזה (לקחים + היסטוריית תוצאות) מוזן חזרה לפרומפט של המנוע הכמותי ומשפיע על התחזית הבאה.

## 3. מדדי דיוק ממומשים (End-to-End)

### מקור הנתונים
- מדדי הדיוק מחושבים **רק מתחזיות שהוערכו** ומאוחסנות ב־`historical_predictions` (כשמשתמשים ב־SQLite עם `DB_DRIVER=sqlite`). ב-Vercel ללא SQLite הבלוק לא יוצג.

### מה מוצג באופס (OpsMetricsBlock)
- **אחוז הצלחה (Win Rate):** אחוז התחזיות שהוערכו כ־"הצלחה" (bullish_win / bearish_win / neutral_win) מכלל התחזיות בהיסטוריה שנטענה.
- **שגיאה ממוצעת (Avg Error):** ממוצע `absolute_error_pct` על כל התחזיות שהוערכו.
- **מגמת דיוק:** השוואה בין **10 התחזיות האחרונות** ל־**10 שקדמו להן**:
  - "10 אחרונות: X% | 10 קודמות: Y%"
  - **↑ משתפר** (ירוק) כאשר אחוז ההצלחה ב־10 האחרונות גבוה מ־10 הקודמות.
  - **↓** (ענבר) כאשר אחוז ההצלחה ב־10 האחרונות נמוך מ־10 הקודמות.

### איפה לראות
- **לוח הבקרה** (`/ops`), מתחת לכפתור "הערך תחזיות ממתינות": בלוק "דיוק ממומש (מתחזיות שהוערכו)" עם אחוז הצלחה, שגיאה ממוצעת ומגמת דיוק (אם קיימות לפחות 20 תחזיות שהוערכו ב־historical_predictions).

---

## 4. סיכום זרימת הלמידה

1. **תחזית נוצרת** → נשמרת כ־pending.
2. **המפעיל לוחץ "הערך תחזיות ממתינות"** → המערכת משווה למחיר נוכחי, קובעת הצלחה/כישלון, קוראת ל-Gemini ליצירת מסקנת למידה, שומרת `error_report` ו־`learning_context` ומעבירה ל־historical_predictions.
3. **המשתמש רואה ב־Insights** את "לקחים שנלמדו / מסקנות AI" לכל תחזית שהוערכה.
4. **בניתוח הבא** המנוע הכמותי מקבל את `past_mistakes_to_learn_from` ו־`historical_prediction_outcomes` ומתאים את ההנמקות והביטחון.
5. **מדדי האופס** מציגים דיוק ממומש ומגמה (10 אחרונות מול 10 קודמות) כהוכחה שהלולאה פעילה ומשפרת.
