# פרוטוקול בדיקות QA — דומיין חי (Production)

**דומיין:** https://www.moncherigroup.co.il  
**מטרה:** בדיקה ידנית מלאה לפני ואחרי פריסה. יש לבצע את הפריטים להלן בסדר המופיע.

---

## 1. טעינה ראשונית ושגיאות קונסול

- [ ] **1.1** פתח את האתר ב־**מצב פרטי/אינקוגניטו** (ללא הרחבות).
- [ ] **1.2** פתח את **Developer Tools** (F12) → לשונית **Console**.
- [ ] **1.3** טען את הדף הראשי (`/`).
- [ ] **1.4** וודא שאין **שגיאות אדומות** (errors) בקונסול. אזהרות (warnings) מותרות.
- [ ] **1.5** וודא שאין הודעות **hydration** (React hydration failed / mismatch).
- [ ] **1.6** רענן את הדף (Ctrl+F5) ובדוק שוב שאין שגיאות חדשות.

**תיעוד:** אם יש שגיאה — צלם מסך והעתק את הטקסט המלא.

---

## 2. אימות והגנת נתיבים

- [ ] **2.1** בעודך **לא מחובר**, גלוש ישירות ל־`https://www.moncherigroup.co.il/ops`.
- [ ] **2.2** וודא שאתה **מנותב אוטומטית** ל־`/login?from=/ops` (או דף לוגין עם פרמטר `from`).
- [ ] **2.3** נסה גישה ישירה ל־`/ops/strategies`, `/ops/pnl`, `/settings` — כולם חייבים להפנות ללוגין (או להחזיר 401/302).
- [ ] **2.4** התחבר עם סיסמת האדמין התקינה.
- [ ] **2.5** וודא שאחרי התחברות מוצלחת אתה מועבר ל־`/ops` (או ל־`from` אם הוגדר, למשל `/ops/strategies`).
- [ ] **2.6** וודא שכל הנתיבים המוגנים נגישים אחרי התחברות: `/ops`, `/ops/strategies`, `/ops/pnl`, `/settings`.
- [ ] **2.7** לחץ על **התנתק** (בכותרת).
- [ ] **2.8** וודא שאתה מנותב ל־`/login` ושאי אפשר לגשת שוב ל־`/ops` ללא התחברות מחדש (נסה לפתוח `/ops` בטאב חדש — חייב להפנות ללוגין).

**תיעוד:** ציין אם ניסיון עקיפת לוגין (נתיב ישיר) נחסם כמצופה.

## 3. WebSocket — זרם שוק (Ticker)

- [ ] **3.1** התחבר ואפשר לדף הראשי או ל־`/ops` להיטען.
- [ ] **3.2** בראש הדף אמור להופיע **פס טיקר** (מחירי מטבעות עם אחוז שינוי).
- [ ] **3.3** וודא שמופיע אינדיקטור חיבור: **חי** (ירוק) או **מתחבר** (כתום). אם מופיע **מתחבר מחדש** (אדום), חכה כמה שניות — אמור להתחבר מחדש אוטומטית.
- [ ] **3.4** ב־**Network** (Developer Tools) בדוק שיש חיבור **WebSocket** (סטטוס 101 או פעיל). אם האתר מאחורי פרוקס/חומת אש, ייתכן חסימה — תתעד.
- [ ] **3.5** (אופציונלי) הפעל **Throttling** (למשל Slow 3G), חכה לניתוק, ואז השבת — וודא שהטיקר חוזר ומציג **חי** ללא קריסת דף.

**תיעוד:** ציין אם הטיקר מציג נתונים והאם לאחר ניתוק יש התחברות מחדש.

## 4. פעולות אסינכרוניות — מצבי טעינה ושגיאה

- [ ] **4.1** בדף **לוגין**: הזן סיסמה שגויה ולחץ כניסה — וודא שמופיעה **הודעת שגיאה** (אדום) והכפתור חוזר למצב רגיל (לא תקוע ב"מאמת…").
- [ ] **4.2** בדף **לוגין**: הזן סיסמה נכונה — וודא שמופיע **מצב טעינה** (ספינר + "מאמת…") עד ההפניה ל־`/ops`.
- [ ] **4.3** ב־**/ops**: לחץ על **"הרץ סימולציית BTC"** — וודא שמופיע **ספינר / "מנתח נתונים…"** והכפתור מושבת, ואז תוצאה (הצלחה או הודעת שגיאה).
- [ ] **4.4** ב־**/ops**: לחץ על **"הרץ ניתוח"** (במודול ניתוח) — וודא **מצב טעינה** ו**הודעת שגיאה או הצלחה** (לא כפתור "מת").
- [ ] **4.5** ב־**תובנות AI** (/insights): בחר תחזית ולחץ **"אשר סימולציה"** — וודא ספינר והודעה (הצלחה או שגיאה).
- [ ] **4.6** ב־**הגדרות** (/settings): לחץ על **"Test Telegram"** או **"בדוק חיבור"** — וודא מצב "שולח..." ותוצאת בדיקה (הצלחה/כישלון).
- [ ] **4.7** לחץ **התנתק** — וודא שמופיע "מתנתק…" (או ספינר) ואז הפניה ל־`/login`.

**תיעוד:** ציין כל כפתור שלא הגיב או שלא הציג טעינה/שגיאה.

## 5. מסד נתונים וסימולציה

- [ ] **5.1** ב־**/ops** הרץ **"הרץ סימולציית BTC"** עד להצלחה (אם יש שגיאת רשת/שרת — תתעד).
- [ ] **5.2** וודא שמופיעה תוצאה עם כיוון, הסתברות וסנטימנט (או הודעת שגיאה ברורה).
- [ ] **5.3** גלוש ל־**תובנות AI** (/insights) וודא שמופיעה רשימת תחזיות (או "אין עדיין תחזיות" אם המסד ריק).
- [ ] **5.4** ב־**/ops** גלול ל-**מדדי מערכת** (OpsMetricsBlock) — וודא שמוצגים מספרים (סה"כ תחזיות, ממתינות, אומתו, זמן תגובה וכו') או הודעה ברורה ("זמן הטעינה פג" / "לא ניתן לטעון מדדים") ללא קריסת דף.

**תיעוד:** ציין אם סימולציה נשמרת ואם מדדי המערכת נטענים.
