# דוח חתימה — מרכז פקודות מאסטר (עמוד הגדרות)

**תאריך:** 2025-03-15  
**גרסה:** 1.0  
**סטטוס:** הושלם ואושר

---

## 1. סיכום ביצוע

בוצע שדרוג מלא לעמוד ההגדרות (עמוד הגדרות) למרכז פקודות מאסטר (Master Command Center) עם ערכת עיצוב "ים עמוק" (Deep Sea), חמש קטגוריות מובחנות, סנכרון מלא עם ה־Backend ורישום ביקורת.

---

## 2. UI/UX ומבנה העמוד (תא הטייס הים עמוק)

- **עיצוב:** העמוד משתמש בערכת "Deep Sea" — רקע כחול־נייבי (`#030f1c`, `#07192d`), גבולות וטקסט בגווני ציאן, כפתורים ואקורדים בציאן.
- **ניווט:** טאבים עליונים (Top Tabs) לחמש קטגוריות:
  - **א. מסחר וביצוע:** גודל עסקה ברירת מחדל ($), מקסימום פוזיציות פתוחות, סובלנות החלקה מקסימלית (%).
  - **ב. ניהול סיכונים:** חשיפה גלובלית מקסימלית (%), מגבלת ריכוז נכס בודד (%), מכפילי ATR ל־TP ו־SL, סטופ־לוס ויעד רווח ברירת מחדל.
  - **ג. בינה מלאכותית:** סף ביטחון MoE (ציון מינימלי לאישור עסקה, ברירת מחדל 75), הפעלת RAG (זיכרון היסטורי), פוסט־מורטם אוטומטי, סף ביטחון סורק AI, נפח מינימלי 24 שעות, שינוי מחיר מינימלי לג'ם.
  - **ד. התראות מנכ"ל:** מתגים — דוח פעימה יומי, התראות סיכון קריטי, ג'ם עלית חדש זוהה, התראות טלגרם, התראות קול, ערכת נושא, מרווח רענון נתונים.
  - **ה. אבטחה ובקרה:** טבלת System Audit (תצוגת קריאה בלבד של `audit_logs`).
- **RTL:** תוויות בעברית מיושרות לימין (RTL); מספרים ומפתחות API ב־LTR.
- **טפסים:** ניהול state עם React Hook Form; כפתור "שמור שינויים" עם טוען מסתובב בעת שמירה; הודעת Toast בהצלחה: "הגדרות המערכת עודכנו בהצלחה"; טיפול בשגיאות מהשרת; אזהרת שינויים שלא נשמרו בעת מעבר טאב.

---

## 3. Backend וסכמה

- **טבלת `settings`:** נשמר JSON תחת המפתח `app_settings`. הסכמה הורחבה לקטגוריות: `trading`, `risk`, `scanner`, `neural`, `notifications`, `system`.
- **`/api/settings/app`:**
  - **GET:** דורש סשן תקף כאשר אימות מופעל; מחזיר את כל ההגדרות (ממוזגות עם ברירות מחדל).
  - **POST:** דורש סשן Admin כאשר אימות מופעל; מאמת סוגי נתונים (מספרים לא כמחרוזות, אחוזים 0–100); משתמש ב־`recordAuditLog` עם `payload_diff` JSON שמכיל רק את השדות שהשתנו.
- **תיעוד ביקורת:** כל שמירת הגדרות נרשמת ב־`audit_logs` עם `action_type: 'settings_update'` ו־`payload_diff` של השינויים בלבד.

---

## 4. חיווי וריאקטיביות

- **כפתור שמור:** מציג טוען (Loader) במהלך השמירה; מושבת כאשר אין שינויים.
- **Toast הצלחה:** "הגדרות המערכת עודכנו בהצלחה" (באמצעות ToastContext הקיים).
- **שגיאות:** הודעת שגיאה ברורה במקרה של כשל API או רשת.
- **אזהרת שינויים:** בעת מעבר לטאב אחר עם שינויים שלא נשמרו — מוצגת הודעת אישור (confirm).

---

## 5. אינטגרציה גלובלית

- **עדכון מיידי לאחר שמירה:** קריאה ל־`refreshSettings()` מ־`AppSettingsContext` לאחר שמירה מוצלחת — כל האפליקציה מקבלת את ההגדרות המעודכנות (כולל ערכת נושא ומרווח רענון).
- **חיבור למנועים:**
  - **סורק / MoE:** סף הביטחון של MoE (`neural.moeConfidenceThreshold`) משמש את מנוע הקונצנזוס; הסורק משתמש ב־`scanner.aiConfidenceThreshold` וב־`trading.defaultTradeSizeUsd` (או `risk.defaultPositionSizeUsd`) לסכום עסקה.
  - **ניהול סיכונים:** `risk.globalMaxExposurePct` ו־`risk.singleAssetConcentrationLimitPct` מועברים ל־Exposure Sentinel; מכפילי ATR (`risk.atrMultiplierTp`, `risk.atrMultiplierSl`) משמשים את ליבת הניתוח (analysis-core).
- **כפתורים:** כל הכפתורים (טאבים, שמור, מתגים) מחוברים ופעילים; אין "כפתורים מתים".
