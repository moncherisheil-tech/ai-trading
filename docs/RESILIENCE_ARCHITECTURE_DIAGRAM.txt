================================================================================
  KEY ARCHITECTURAL FEATURES
================================================================================

1. EXPONENTIAL BACKOFF WITH JITTER
   ┌─────────────────────────────────────────────┐
   │ delay = 2^attempt * 1000ms + random(0, 500) │
   │                                             │
   │ Prevents Thundering Herd:                   │
   │ • Attempt 1 @ 0-500ms delay                 │
   │ • Attempt 2 @ 2-2.5s delay                  │
   │ • Attempt 3 @ 4-4.5s delay                  │
   │                                             │
   │ If all attempts fail → is_fallback: true    │
   └─────────────────────────────────────────────┘

2. STRICT OBSERVABILITY
   ┌─────────────────────────────────────────────┐
   │ Every expert output includes:               │
   │                                             │
   │ is_fallback: false                          │
   │   → API call succeeded                      │
   │   → Score is reliable                       │
   │   → Include in weighted average             │
   │                                             │
   │ is_fallback: true                           │
   │   → API key missing OR all retries failed   │
   │   → Score = 50 (neutral fallback)           │
   │   → EXCLUDE from weighted average           │
   └─────────────────────────────────────────────┘

3. DYNAMIC WEIGHT REDISTRIBUTION
   ┌─────────────────────────────────────────────┐
   │ TRADITIONAL APPROACH (BROKEN):              │
   │ final = sum(score × weight) / sum(weight)   │
   │ Problem: Dead API always included           │
   │                                             │
   │ NEW APPROACH (FIXED):                       │
   │ Filter: experts where !is_fallback          │
   │ final = sum(score × weight for live)        │
   │       / sum(weight for live)                │
   │ Benefit: Dead expert = 0 impact             │
   └─────────────────────────────────────────────┘

4. INTEGRATED OBSERVABILITY
   ┌─────────────────────────────────────────────┐
   │ ConsensusResult now includes:               │
   │                                             │
   │ • tech_fallback_used?: boolean              │
   │ • risk_fallback_used?: boolean              │
   │ • psych_fallback_used?: boolean             │
   │ • macro_fallback_used?: boolean             │
   │ • onchain_fallback_used?: boolean           │
   │ • deep_memory_fallback_used?: boolean       │
   │                                             │
   │ Operators can see:                          │
   │ ✓ Which experts failed                      │
   │ ✓ How many fallbacks (board health)         │
   │ ✓ Whether consensus is corrupted            │
   └─────────────────────────────────────────────┘
