## 2. מנוע הכיול האוטונומי (`lib/optimizer.ts`)

### 2.1 ניתוח רגישות (Sensitivity Analysis)

- **נתונים:** עסקאות היסטוריות מ-14 הימים האחרונים (בקטסט + תיק וירטואלי).
- **סימולציות What-If:** הזזת פרמטרי TP, SL ונפח (Volume) בצעדים של 5% ו-10% (מכפילים: 0.85–1.15).
- **יעד אופטימיזציה:** מקסום **יחס שרפ** (Sharpe Ratio):
  $$S = \frac{E[R_a - R_b]}{\sqrt{Var(R_a - R_b)}}$$
  כאשר \(R_b = 0\) (שיעור חסר סיכון בסימולציה).
- **תוצאה:** זיהוי **נקודת איזון אופטימלית** — פרמטרים מוצעים ל-TP, SL וגודל פוזיציה.

### 2.2 API כיול — `GET /api/ops/calibrate`

- מחזיר המלצות פעולה והקשר שוק (Market Context).
- **דגל הקשר שוק:** `normal` | `high_volatility` | `low_volatility`.
- **לוגיקת מומחה:** במצב תנודתיות גבוהה — הורדת TP בכ־15% משפרת היסטורית את מקדם הרווח (Profit Factor) ומפחיתה חשיפה לסיכון.

## 3. מרכז אופטימיזציה (Optimizer Hub) — אינטגרציה בממשק

### 3.1 כרטיס "כיול מערכת אוטונומי"

- **מיקום:** דף הגדרות (`/settings`) ומרכז אנליטיקה (`/analytics`).
- **תוכן:** הצגת **פרמטרים נוכחיים** מול **פרמטרים מוצעים** (TP, SL, גודל פוזיציה).
- **כפתור "החל סכימת כיול":** מעדכן את `AppSettings` (risk) באופן מיידי דרך `POST /api/settings/app`.
- **טרמינולוגיה:** ניתוח רגישות, אופטימיזציית פרמטרים, נקודת איזון אופטימלית — בעברית ובממשק.

## 4. בקרת איכות (QC) ואימות מתמטי

### 4.1 יושר מתמטי — Decimal.js ומקדם רווח (Profit Factor)

- **נוסחה מאומתת:**
  $$\text{Profit Factor} = \frac{\sum \text{Gross Profits}}{\sum \text{Gross Losses}}$$
- **טיפול בתקופות ללא הפסד:** אם סך ההפסדים = 0: מקדם רווח = 999 אם קיים רווח, אחרת 0.
- **מניעת NaN ואינסוף:** כל חישובי PF ו-Sharpe עוברים בדיקת `Number.isFinite()` ועיגול באמצעות `round2`.
- **מיקומים:** `app/api/ops/metrics/historical/route.ts`, `app/api/ops/metrics/pnl/route.ts`, `lib/optimizer.ts`.

### 4.2 סנכרון זרימת נתונים — מרכז למידה ו-CEO Briefing

- **טווח תאריכים:** ביצועי תקופה (Historical) ו-CEO Briefing משתמשים באותו `from_date` / `to_date` ממרכז האנליטיקה.
- **API תובנות:** `GET /api/agent/insights` תומך כעת ב-`from_date` ו-`to_date` (אופציונלי) — כאשר מסופקים, מוחזרות תובנות באותו טווח (סנכרון לוגי עם CEO Briefing).
- **ממשק:** תצוגת "תובנות סוכן באותו טווח" וספירת תובנות בתקופה — מסונכרנת ויזואלית עם סיכום התקופה.

### 4.3 אבחון סורק (Scanner Diagnostics)

- **בריאות סורק:** הסבר מדויק מדוע אין ג'מים:
  - **נכשלו בניתוח (API/מודל)** — כישלון בניתוח.
  - **סוננו עקב הסתברות נמוכה או תנאי RSI/שוק (מתחת לסף ביטחון)** — במקום רק "Failed".
  - **כבר קיבלו התראה לאחרונה** — מניעת ספאם.
- **מיקום:** `lib/workers/market-scanner.ts` — פונקציה `buildScannerDiagnosticsSummary`.

## 5. עברית, RTL ועיצוב (Deep Sea)

- **טרמינולוגיה:** ניתוח רגישות, אופטימיזציית פרמטרים, נקודת איזון אופטימלית, כיול מערכת אוטונומי, החל סכימת כיול — נוספו ל-`lib/i18n.ts` ועודכנו בממשק.
- **RTL:** `dir="rtl"` וטקסט בעברית בכל הרכיבים הרלוונטיים (הגדרות, אנליטיקה, כרטיס כיול).
- **ערכת Deep Sea:** נושא `deep-sea` זמין בהגדרות; יישום עקבי של משתני CSS ו-RTL במסוף ברמת מוסדית.
